# UniFi MiniDash v2.0.0 - Design Specification

**Data:** 2026-04-09
**Autor:** Lukasz Misiura / LM-Networks
**Status:** Zatwierdzony

---

## Podsumowanie

Wersja 2.0.0 to duza aktualizacja MiniDash wprowadzajaca:
1. Migracje z JSON na SQLite jako centralny storage danych
2. Wi-Fi Stalker - sledzenie roamingu urzadzen WiFi
3. Threat Watch ulepszenia - ignore list IP, filtry czasu, auto-purge
4. Network Pulse drilldown - klikalne AP z lista klientow
5. Webhooks Discord + n8n
6. Szyfrowanie credentials w config.json
7. Footer z wersja i brandingiem

**Design:** Utrzymany istniejacy styl - glassmorphism, dark theme, Tailwind CSS, Lucide icons, polskie UI.
**Stack:** PHP 8.x, vanilla JS, SQLite (PDO), bez frameworkow.

---

## 1. Warstwa SQLite + migracje

### Nowe pliki

- `db.php` - centralna warstwa bazy danych, include na gorze stron (obok config.php)
- `migrations/` - folder z plikami SQL
- `migrate_json.php` - jednorazowy skrypt migracji JSON -> SQLite
- `data/minidash.db` - plik bazy SQLite (w .gitignore)

### Schemat

```sql
-- System migracji
CREATE TABLE migrations (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    version TEXT NOT NULL UNIQUE,
    name TEXT NOT NULL,
    applied_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- Migracja istniejacych danych z JSON
CREATE TABLE login_history (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username TEXT NOT NULL,
    ip TEXT,
    user_agent TEXT,
    logged_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE wan_stats (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    download_mbps REAL,
    upload_mbps REAL,
    latency_ms REAL,
    recorded_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE client_history (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    mac TEXT NOT NULL,
    hostname TEXT,
    ip TEXT,
    network TEXT,
    vlan INTEGER,
    is_wired INTEGER DEFAULT 0,
    rx_bytes INTEGER,
    tx_bytes INTEGER,
    uptime_sec INTEGER,
    seen_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE device_monitors (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    mac TEXT,
    name TEXT NOT NULL,
    host TEXT NOT NULL,
    enabled INTEGER DEFAULT 1,
    added_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE events (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    type TEXT NOT NULL,
    severity TEXT DEFAULT 'INFO',
    message TEXT,
    details_json TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- Wi-Fi Stalker
CREATE TABLE stalker_sessions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    mac TEXT NOT NULL,
    hostname TEXT,
    ap_mac TEXT,
    ap_name TEXT,
    ssid TEXT,
    channel INTEGER,
    band TEXT,
    rssi INTEGER,
    rx_rate REAL,
    tx_rate REAL,
    connected_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    disconnected_at DATETIME
);

CREATE TABLE stalker_roaming (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    mac TEXT NOT NULL,
    hostname TEXT,
    from_ap TEXT,
    to_ap TEXT,
    from_channel INTEGER,
    to_channel INTEGER,
    rssi_before INTEGER,
    rssi_after INTEGER,
    roamed_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE stalker_watchlist (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    mac TEXT NOT NULL UNIQUE,
    label TEXT,
    notify INTEGER DEFAULT 1,
    blocked INTEGER DEFAULT 0,
    added_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- Threat Watch
CREATE TABLE threat_ignore (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    ip TEXT NOT NULL UNIQUE,
    label TEXT,
    reason TEXT,
    added_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- Indeksy
CREATE INDEX idx_client_history_mac ON client_history(mac);
CREATE INDEX idx_client_history_seen ON client_history(seen_at);
CREATE INDEX idx_wan_stats_recorded ON wan_stats(recorded_at);
CREATE INDEX idx_stalker_sessions_mac ON stalker_sessions(mac);
CREATE INDEX idx_stalker_roaming_mac ON stalker_roaming(mac);
CREATE INDEX idx_stalker_roaming_time ON stalker_roaming(roamed_at);
CREATE INDEX idx_events_created ON events(created_at);
CREATE INDEX idx_events_severity ON events(severity);
```

### db.php - zachowanie

- Tworzy/otwiera `data/minidash.db` przez PDO
- Przy starcie: sprawdza tabele `migrations`, odpala brakujace pliki z `migrations/`
- Auto-purge: raz na sesje sprawdza i czysci stare rekordy:
  - `wan_stats` > 90 dni
  - `client_history` > 30 dni
  - `events` > 30 dni
  - `stalker_sessions` > 60 dni
  - `stalker_roaming` > 60 dni
- Progi purge konfigurowalne w config.json pod kluczem `purge_days`
- Eksportuje globalny `$db` (PDO instance)

### migrate_json.php

- Czyta: `data/login_history.json`, `data/wan_stats.json`, `data/last_clients.json`, `data/devices.json`, `data/history.json`
- Wstawia do odpowiednich tabel SQLite
- Idempotentny - sprawdza czy dane juz istnieja przed insertem
- Loguje postep do konsoli / logow
- Dostepny z przegladarki (wymaga zalogowania) lub CLI

## 2. Wi-Fi Stalker

### Dashboard widget (index.php)

Nowy kafelek w gornym rzedzie obok istniejacych (Aktywni, Sieci WiFi, Sprzet, Ingress, Egress):

- Ikona: Lucide `wifi` lub `radar`
- Tytul: "ROAMING"
- Glowna liczba: zdarzenia roamingu w ostatnich 24h
- Podtytul: ostatni roaming (nazwa urzadzenia, skad -> dokad, czas temu)
- Link: "SZCZEGOLY >" -> otwiera stalker.php

### Strona stalker.php

**Nawigacja:** Nowa ikona w gornym navbar (obok Protect, Security, Logs).

**Gorny pasek filtrow:**
- Szukaj (MAC/nazwa)
- Filtr czasu: 1h / 24h / 7d / 30d
- Filtr pasmo: Wszystkie / 2.4 GHz / 5 GHz / 6 GHz
- Przycisk: Eksport CSV
- Przycisk: Watchlist (otwiera modal)

**Sekcja "Aktywne sesje WiFi"** - tabela w stylu "Aktywni Klienci Sieci":
- Kolumny: URZADZENIE | AP | PASMO/KANAL | RSSI | RX/TX | CZAS POLACZENIA | AKCJA
- RSSI z kolorowym wskaznikiem: zielony > -50, zolty -50..-70, czerwony < -70
- AKCJA: ikona oka (historia w modal), ikona blokady (block/unblock z potwierdzeniem)

**Sekcja "Historia roamingu"** - timeline w stylu logs.php:
- Wpisy: ikona + nazwa urzadzenia + skad AP -> dokad AP + RSSI + czas
- Kolory: zielony = polaczenie, czerwony = rozlaczenie, zolty = roaming
- Paginacja: 25/50/100 na strone

**Sekcja "Watchlist"** - modal:
- Formularz dodawania: MAC + etykieta
- Tabela: MAC | ETYKIETA | POWIADOMIENIA (toggle) | ZABLOKOWANY (toggle) | AKCJA (usun)

### Backend

- `api_stalker.php` - endpoint AJAX:
  - GET: pobiera aktywne sesje i historie roamingu z SQLite
  - POST action=poll: pobiera /stat/sta z kontrolera, porownuje z SQLite, wykrywa roaming, zapisuje
  - POST action=block/unblock: wysyla /cmd/stamgr do kontrolera
  - POST action=export: generuje CSV
- Polling: JS co 30s wywoluje poll, odswieza tabele
- Wykrywanie roamingu: porownanie ap_mac z ostatnia sesja dla danego mac w stalker_sessions

## 3. Threat Watch ulepszenia (security.php)

### IP Ignore List

- Nowy przycisk w gornym pasku security.php: "Ignorowane IP"
- Modal z tabela: IP | ETYKIETA | POWOD | DODANO | AKCJA (usun)
- Formularz dodawania: IP + etykieta + powod
- API: `api_threat_ignore.php` (GET lista, POST dodaj, DELETE usun)
- Mapa zagrozen i lista zablokowanych pomija IP z ignore list
- Mozliwosc dodania IP bezposrednio z mapy zagrozen (ikona "ignoruj" przy kazdym IP)

### Filtry czasu

- Gorny pasek security.php - przyciski zakresu czasu:
  - [ 24h ] [ 7d ] [ 30d ] [ Wszystko ]
- Filtruje: mape zagrozen, statystyki IPS, liste zablokowanych
- Domyslnie: 24h
- Styl: identyczny jak filtry w Aktywni Klienci (przyciski z aktywnym stanem)

### Auto-purge

- Automatyczny z db.php - eventy security > 30 dni czyszczone
- Bez dodatkowego UI - dzialanie w tle

## 4. Network Pulse - drilldown AP -> klienci

### Modyfikacja panelu "Urzadzenia UniFi" (index.php)

- Klikniecie na wiersz urzadzenia (AP/switch) -> rozwija pod spodem liste klientow
- Zaladowane AJAX po kliknieciu (nie preload)
- Dane: /stat/sta filtrowane po ap_mac
- Kolumny zagniezdzonej tabeli: NAZWA | PASMO/KANAL | RSSI | PREDKOSC | SIEC
- Styl: zagniezdzona tabela, lekko jasniejsze glassmorphism tlo
- Klik ponowny na wiersz -> zwija

## 5. Webhooks Discord + n8n

### Nowe kanaly powiadomien

**Discord:**
```php
'discord_notifications' => [
    'enabled' => false,
    'webhook_url' => '',
    'username' => 'MiniDash'
]
```
- Format: Discord embeds (kolorowe karty z polami, kolor wg severity)
- Funkcja: `sendDiscordNotification()` w config.php

**n8n:**
```php
'n8n_notifications' => [
    'enabled' => false,
    'webhook_url' => ''
]
```
- Format: surowy JSON payload z polami: type, severity, message, timestamp, source
- Funkcja: `sendN8nNotification()` w config.php

### UI

- Dwa nowe panele w settings_notifications.php
- Identyczny styl jak istniejace (Telegram, Slack, ntfy)
- Discord: pole URL + pole username bota
- n8n: pole URL

### Integracja

- Dodane do `sendAlert()` w config.php obok istniejacych kanalow
- Stalker moze wysylac alerty przez te kanaly (roaming watchlist)

## 6. Szyfrowanie credentials

### Nowy plik crypto.php

- Generuje klucz szyfrowania przy pierwszym uzyciu
- Klucz zapisany w `data/.encryption_key` (dodane do .gitignore)
- Algorytm: `sodium_crypto_secretbox` (wbudowane PHP 8, libsodium)
- Szyfruje/deszyfruje transparentnie

### Pola szyfrowane w config.json

- `api_key`
- `admin_password`
- Wszystkie tokeny/klucze powiadomien (bot_token, api_key, smtp_password, webhook_url)

### Format

- Zaszyfrowane pole: `"api_key": "ENC:base64encodedciphertext..."`
- Prefix `ENC:` rozpoznaje czy pole juz zaszyfrowane
- `load_app_config()` automatycznie deszyfruje przy odczycie
- `api_save_settings.php` automatycznie szyfruje przy zapisie

### Migracja

- Przy pierwszym uruchomieniu z crypto.php - automatycznie szyfruje istniejace plaintext pola
- Jednorazowa operacja, transparentna

## 7. Footer

### Nowy plik includes/footer.php

Include na dole kazdej strony (index.php, stalker.php, security.php, logs.php, protect.php, monitored.php, history.php, settings_notifications.php, devices.php).

**Layout:** Flexbox, space-between:

```
LM-Networks (c) 2026 · Wszelkie prawa zastrzezone                    MiniDash v2.0.0
Czesc ekosystemu LuMiGRAF Solutions
```

- Lewa strona: branding + copyright + opis ekosystemu (mniejszy tekst pod spodem)
- Prawa strona: wersja aplikacji

**Styl:**
- Subtleny, polprzezroczysty tekst (text-gray-500/60)
- Padding 4-6, max-w-7xl (jak reszta contentu)
- Wersja pobierana z globalnej stalej `MINIDASH_VERSION` zdefiniowanej w config.php
- Glassmorphism border-top subtelny
